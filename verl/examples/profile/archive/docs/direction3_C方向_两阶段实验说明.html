同组 4 个 branch 的 turn1 refocus实验说明

实验目标

GRPO 里 rollout_n=4，同一个 prompt + 同一张原图会开出 4 条 branch。每条 branch 走完 refocus 后都会有一张 turn1 新图。这 4 张 turn1 图彼此像不像？像到 ViT 的 image token 能不能在 group 里互相复用？

两个阶段的区别

阶段一：原始 refocus + Oracle 阶段二：Diversified Oracle（受控改造）
目的 用数据集自带的 确定性 oracle refocus，模拟「正常 profiling 跑 refocus」 故意让 4 个 branch 框不同区域，且 只用 draw，测「分化最大」时 token 还能不能复用
4 个 branch 的 turn1 理论上应跑 同一段 oracle → 多数组 4 张图字节级相同 每 branch 不同 bbox，刻意 4/4 不同图
意义 乐观上界（exact dedup 能命中多少） 保守下界（最不像的时候 cos 还有多少）
数据目录 profile_logs_refocus_chart_origin/ profile_logs_refocus_chart/（VTOOL_ORACLE_DIVERSIFY=1）

阶段一：原始 refocus + Oracle

方法论

  1. 设定：TRAIN_BATCH_SIZE=64，ROLLOUT_N=4，Refocus Chart 多轮数据；agent 开 force_oracle_refocus，用数据里写好的 oracle_refocus_code（focus_on_*_with_draw/highlight/...），不是让 policy 自己学出来的 tool call。
  2. 一个 group 里发生什么：
  3. turn0：4 个 branch 输入 同一张原图（chart）。
  4. turn1：每个 branch 执行 oracle 做 refocus，得到 refocus 输出图。
  5. 怎么量：
  6. 对每组 4 张 turn1 refocus PNG，用 Qwen2.5-VL-7B 的 ViT 重新 encode，得到每张图的 image token 序列（长度 = n_tokens）。
  7. 两两算 per-token cosine，汇总 token_mean_cos、reusable@0.99（cos≥0.99 的 token 占比）等。

Image token 数量

场景 含义 本组数值

本例小结（阶段一）：即使 oracle 本意是「大家做同一种 refocus」，这个组里因为 执行结果不一致（失败 / 多框组合不同），4 张 turn1 已经不一样了，但 平均仍有 ~90% 的 token cosine 很高、约一半 token 达到 0.99 可复用阈值——说明该实验的 partial reuse 在「不同 refocus 图」之间是存在的。

阶段二：Diversified Oracle（受控改造）

方法论

  1. 为什么要改造：阶段一里 95% 的组 turn1 完全一样，该实验回答不了「4 条 branch 若 refocus 成不同局部图，token 还能复用吗」。
  2. 我们改了什么（VTOOL_ORACLE_DIVERSIFY=1 + vtool_agent_loop_diversified.yaml）：
  3. 仍用 oracle 管线，但按 branch 的 request_id 哈希，从 metadata 的 bbox 列表里 选不同 key；
  4. 固定只用 draw（画红框），不用 mask（mask 会把大半张图涂白，和「局部框选」不是一类编辑）；
  5. 不声称这是真实 RL policy 的自然行为 → 定位成 controlled synthetic local-edit workload。
  6. 比较对象仍然一样：比 4 张 turn1 的 ViT token cosine；只是现在 刻意让 4 张图尽量不同（保守、分化最大的情形）。

结论

来自 profile_logs_refocus_chart/similarity_diversified/similarity_summary_by_pair_type.md：

指标 数值 和阶段一对比
turn1 跨 branch byte-identical 100 / 585 ≈ 17% 从 95% 降到 17%，真分化了
token mean cos（全部 turn1 对） 0.9125 仍很高，但低于阶段一的 0.9955
只统计「图确实不同」的 485 对 mean cos 0.8944，reusable@0.99 49.4% 这才是保守的
reusable@0.95（全部） 74.8% 约 3/4 token 仍可当 partial cache 候选

结论（阶段二）：故意让 4 个 branch 框不同区域、只画框，turn1 大多已是不同 PNG，但 token 仍高度相似（~0.89–0.91）—— group-level partial image-token reuse 在保守设定下仍成立。
